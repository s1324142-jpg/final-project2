# 仕様メモ

## データの流れ

1. フロントエンドで言語とソースコードを入力する。
2. `POST /api/reviews` に送信する。
3. Express サーバーが OpenAI Responses API にレビュー依頼を送る。
4. AI は JSON Schema に従って、分析、改善コード、評価を返す。
5. サーバーは結果を `data/reviews.json` に保存する。
6. フロントエンドは分析、生成、評価、履歴の各画面へ結果を描画する。

## AI プロンプト方針

- 学習者向けにやさしい日本語で説明する。
- 元のコードの意図を保ち、過度な書き換えを避ける。
- 指摘は具体的な問題、重要度、説明、改善提案に分ける。
- セキュリティ指摘はコードから判断できる範囲に限定する。
- 出力は画面表示用の固定 JSON 構造にする。

## 保存形式

レビュー履歴は `data/reviews.json` に最大 30 件保存します。各履歴は以下を持ちます。

- `id`
- `source`
- `language`
- `originalCode`
- `createdAt`
- `review`
